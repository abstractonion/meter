# Visual Hierarchy

One focal point per viewport. Everything else supports or recedes.

## The squint test

Blur your eyes or squint at the layout. You should still see:

1. **Primary** — one dominant block (headline, key metric, primary CTA)
2. **Secondary** — supporting context
3. **Tertiary** — metadata, chrome

If squint reveals five equals, hierarchy failed.

## Size × weight × color × position

Hierarchy tools stack — don't rely on one:

| Tool | Strong move | Weak move |
| --- | --- | --- |
| Size | 2+ scale steps | 1px difference |
| Weight | 400 → 600 | 500 → 600 alone |
| Color | `fg` vs `fg-muted` | Slightly different gray |
| Position | Top-left / center hero | Buried below fold |

## F-pattern & Z-pattern

- **F-pattern:** Text-heavy pages — users scan top, then down left edge. Put key actions on the left rail or top bar.
- **Z-pattern:** Marketing — logo → CTA diagonal → bottom action. Don't fight the Z with a sidebar.

## Above the fold

Mobile: assume **one** primary action visible without scroll. Desktop: hero + nav + one CTA minimum.

Don't hide the only action below three paragraphs of mission statement.

## Grouping beats boxing

Related items closer (Gestalt proximity) before adding borders. Increase gap between groups, decrease within groups.
